# Astro Migration Regression Checklist

This checklist captures the legacy Vite/React SPA behavior that the Astro content-site migration must preserve or intentionally defer.

## Legacy data baseline

- Legacy resources: 204 entries from `src/data/resources.json`.
- Legacy groups: 9 entries from `src/data/groups.json`.
- Legacy statuses: `curated`, `pending`.
- Legacy type labels: 64 distinct values.

## Feature checklist

| Feature | Legacy behavior | Astro acceptance criterion |
| --- | --- | --- |
| Landing page | `/` opens the atlas landing experience with brand, hero copy, resource statistics, and entry CTA. | `/` renders indexable hero copy, brand identity, and collection-derived resource/group counts matching migrated content. |
| Resource list | Main catalog renders all resources and supports browsing by group and type. | `/resources` renders 204 migrated resources, links each item to a detail page, and exposes group/type facets derived from content. |
| Group filtering | Selecting a group scopes visible resources to resources whose legacy `group` matches that group. | Filtering or navigating to a group page returns the same resource set as the legacy JSON group match. |
| Type filtering | Selecting a type filters resources within the current scope. | Type filtering uses migrated normalized/legacy type metadata and returns the same resources as legacy type equality where that type exists. |
| Fuse.js search | Search matches resource name, group, type, and use text with highlighted matches. | Pagefind search indexes generated resource/group/tag/detail text; searching representative resource names and use text returns the matching detail/listing pages. |
| Resource detail drawer | Clicking a resource opens an in-page drawer with favicon, name, group, type, status, use text, URL, and related resources. | `/resources/[slug]` renders equivalent metadata as a shareable static page; the drawer interaction itself is deferred unless explicitly restored. |
| Related items | Detail drawer suggests resources sharing type and/or group. | Resource detail pages show deterministic related resources scored by shared group, type, tags, and available metadata. |
| Bilingual UI | UI labels can switch between Chinese and English vocabulary. | Static labels and React islands use preserved Chinese/English vocabulary where practical; core browsing labels remain understandable in both languages. |
| Keyboard shortcuts | SPA supports search focus, random/surprise resource, navigation help, and Escape-style modal/drawer closing. | First Astro migration preserves keyboard shortcuts only for hydrated interactions selected for the release; unsupported legacy shortcuts are listed as deferred. |
| Favicon display | Resource rows/cards show fetched favicon assets from `public/icons/` with fallback. | Resource listings and detail pages show favicon assets keyed by migrated content slug or a visible fallback. |
| Pending/curated status | Curated resources are primary; pending resources are visually subdued and counted separately. | Migrated content preserves `status`; listings and detail pages expose curated/pending semantics and counts. |
| Vercel deployment | Static Vite build deploys with Vercel using `dist`. | Astro static build outputs to `dist`, Pagefind indexes production HTML, and Vercel deployment docs/config reference `bun run build`. |

## Non-blocking closeout notes

- `bun run check:links` is a maintenance command because public-site availability depends on external hosts; live-network failures are tracked, not release-blocking.
- Screenshot generation remains optional follow-up work from the original brief. The script exists, but a real batch requires Playwright browser installation in the executing environment.
- Vercel/Cloudflare compatibility is documented and build-verified locally; an actual remote deployment is an operational follow-up, not a code-completeness prerequisite.
